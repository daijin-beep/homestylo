# HomeStylo V2 技术方案

> 版本：V2.0 | 日期：2026年3月 | 状态：架构确认，待实施
> 
> 基于前序版本（V1 Route D/E）的失败经验、两轮深度技术调研、以及与产品负责人的架构讨论。

---

## 一、核心设计原则

### 1.1 从 V1 失败中学到的教训

V1（Route D/E）的根本问题：**"先贴图后修"路线在用户照片场景下不可行。**

- 商品图是正面平视角，用户照片是任意透视角——视角永远对不上
- viewAngleMatcher 用 FLUX Kontext 尝试"旋转"商品图，技术上不可行，高概率 fallback 回原图
- Route E 的"家具像素100%不变"原则锁死了错误的角度和光照
- inverseMaskGenerator 实现偏离设计（只生成边缘环而非全环境反向），但即便修正 mask 也无法解决上述根本矛盾

### 1.2 V2 的三条铁律

**铁律一：直接在用户原图上操作，不生成新视角。**
用户看到的底图就是自己拍的照片——天然可信，不存在"这不是我家"的信任危机。

**铁律二：尺寸精确度由几何保证，AI 不决定任何空间参数。**
家具在效果图中占多少像素，由 3D 投影矩阵的确定性数学计算决定。AI（FLUX Fill Pro）只负责在精确的 mask 区域内画出好看的家具。

**铁律三："放不放得下"的判断优先于"好不好看"的效果。**
5秒内先出俯视图（纯数学、瞬时），让用户立刻看到尺寸关系。20-30秒后出 AI 效果图。
